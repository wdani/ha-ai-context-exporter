# Review Bundle

## Summary

- Treated this as the follow-up task `separate-active-mode-and-transition-status-in-ingress`.
- Inspected the current 0.0.18 merged ingress masking-status implementation before changing code.
- Confirmed the merged baseline already had stable, reconnecting, and reconnected frontend lifecycle states, backed by read-only `/api/info` polling.
- Identified the remaining UX gap: the active masking mode and reconnect/recovered lifecycle all rendered through one status paragraph, so the page still felt like one box swapping text.
- Split the ingress masking status into a persistent active-mode block and a separate temporary transition-status block.
- Kept the active-mode block visible independently from transition messaging; when the backend is temporarily unavailable, it shows the last confirmed mode instead of pretending to have fresh data.
- Made the recovered state more noticeable by showing a separate reachable-again block that confirms the now-active mode and can contrast the previous active mode from frontend runtime memory when available.
- Kept the implementation frontend-only with no backend status payload change, export payload behavior change, masking behavior change, export category change, or warning structure change.
- Did not add a writable settings UI, config mutation endpoint, Supervisor restart control logic, POST/PUT/PATCH/DELETE behavior, service calls, WebSocket access, raw attributes, relationship modeling, export redesign, or persistence beyond page runtime memory.
- Bumped both `ha_ai_context_exporter/rootfs/app/version.py` and `ha_ai_context_exporter/config.yaml` from `0.0.18` to `0.0.19` because this step adds a real user-visible UX structure improvement.

- This UX change cannot prevent, suppress, or replace Home Assistant Supervisor-level restart conflict messages such as another restart job already running.
- The export page only displays and refreshes the current add-on option state; users still change `allow_sensitive_values` in the Home Assistant add-on configuration.
- Masking remains best-effort only and users must manually review exports before sharing them.
- No browser runtime or Home Assistant ingress restart test was run in this local environment.

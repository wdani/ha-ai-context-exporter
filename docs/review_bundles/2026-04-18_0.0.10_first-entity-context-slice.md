# Review Bundle

## Summary

- Implemented the first compact Entity Context slice under the existing `entities` export category.
- Kept the existing `entities` summary fields intact: `status`, `reason`, `reachability`, `readability`, `count`, and `top_domains`.
- Added `entities.items` only when `/states` is readable through the existing local GET-only Core Proxy path.
- Each item is intentionally small: `entity_id`, derived `domain`, direct `state`, and direct plain-string `friendly_name` from `attributes.friendly_name`.
- Missing or non-string `friendly_name` values are exported as `null`; no other attributes are exported.
- Items are sorted by `entity_id` before a fixed hard cap of 50 entries is applied.
- The 50-item cap is an intentional first-slice limitation and must be reevaluated when Entity Context moves beyond this compact slice or mode-specific entity depth is introduced.
- No relationship logic, raw/full attributes, dashboard work, areas/devices work, integrations work, YAML/file parsing, analyzer logic, or export version change was added.
- Bumped the application/add-on version from `0.0.9` to `0.0.10` for the functional Entity Context improvement.

# Review Bundle

## Summary

- Investigated whether Areas / Devices discovery can gain one tiny safe fallback after `0.0.8`.
- No alternate GET-compatible Areas / Devices path could be validated cleanly inside the current local backend architecture.
- Kept the backend conservative: no WebSocket handling, no RPC bridge, no POST/template fallback, no relationship inference, and no speculative candidate probing.
- Documented the limitation honestly and left the application version at `0.0.8`.
